The three key facts

Draft content, server logs, training data.

Three questions people ask most about data privacy. Here's the direct answer to each.

Draft content: not stored

The agreement you draft is built and edited in your browser tab. The text of your draft is not saved to our database, not shared with any third party, and not retained after your session ends. Close the tab — it's gone from our side.

Server logs: yes, standard

Like any web service, BizDraft collects standard server logs — IP address, browser type, page visited, time of visit. These are used to keep the service running and improve the product. They are not tied to the content of your draft.

Training data: no

The content of your draft is not used to train the AI model. It is not shared with any model provider for training purposes. Full details are in the privacy notice.

What gets collected

The full data picture.

When you use BizDraft, here is what happens with data. The drafting model runs server-side — the text you type during the session is sent to it the same way any web request is sent to any web service. Once your session ends, those request logs follow standard short-retention rules. The content isn't stored indefinitely.

Standard analytics (page views, session duration, agreement types used) are collected for product improvement. This is the same analytics infrastructure used by virtually every web product. None of it is tied to your draft content.

BizDraft does not require you to create an account, which means there is no profile, no email address, and no persistent identity attached to your session. You are anonymous to the extent any web user is anonymous — the server log has an IP address, not a name.
